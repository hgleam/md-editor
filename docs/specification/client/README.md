# md-editor / mdview — 依頼者向け仕様書

最終更新: 2026-07-10

## 概要

指定した Markdown ファイルを既定ブラウザで表示する CLI ツール `mdview`。md の親フォルダをローカル HTTP サーバで公開するため、`.md` 同士の相対リンクや画像を辿って回遊できる。mermaid のコードブロックは図として描画される。

## 主要機能

| 機能 | 説明 | 状態 |
|------|------|------|
| Markdown 表示 | `mdview <file.md>` で既定ブラウザに描画表示 | ✅ 実装済み |
| mermaid 描画 | ` ```mermaid ` コードブロックを図として描画 | ✅ 実装済み |
| コードハイライト | highlight.js。配色モード（ライト/ダーク）で自動テーマ切替 | ✅ 実装済み |
| 相対リンク回遊 | `.md`/画像への相対リンクをサーバ経由で解決し、リンク先 md もレンダリング表示 | ✅ 実装済み |
| ドラッグ&ドロップ / ファイル選択 | ブラウザ上で別の md を開き直す（ヘッダの「ファイルを開く」・D&D） | ✅ 実装済み |
| アイドル自動終了 | 一定時間アクセスが無ければサーバを自動終了 | ✅ 実装済み |

## 利用方法

グローバルコマンドとして使う（一度だけ）:

```bash
npm link      # `mdview` を PATH に登録
```

表示:

```bash
mdview path/to/file.md
```

- 既定ブラウザで `http://127.0.0.1:<port>/<file>.md` が開く。
- ページ内の相対リンク（`./other.md`・画像）を辿ると、リンク先もレンダリング表示される。
- 終了は起動したターミナルで **Ctrl-C**、または一定時間アクセスが無ければ自動終了。

link せずに使う場合:

```bash
node bin/mdview.mjs path/to/file.md
```

## 環境変数

| 変数名 | 用途 | 必須 |
|--------|------|------|
| `MDVIEW_IDLE_MIN` | アイドル自動終了までの分数。`0` で無効。既定 `30` | No |

## 制約・受け入れ基準

- mermaid / highlight.js / marked を **CDN から読み込む**ため、表示時にネット接続が必要。
- サーバは `127.0.0.1`（ローカルのみ）で待ち受ける。外部公開はしない。
- 表示対象 md の**親ディレクトリ配下のみ**を配信する（配下外のファイルへのリンクは辿れない＝403）。
- `mdview <file.md>` は既存ファイルを 1 つ引数に取る。ディレクトリ・複数指定は非対応。
